¿Qué es HTML?

HTML es el lenguaje de marcado estándar para crear páginas web.

  • HTML significa Hyper Text Markup Language, Lenguaje de Marcado de Hyper Texto
  • HTML describe la estructura de las paginas Web usando marcado
  • Los elementos HTML son las piezas con las que se construyen las paginas HTML
  • Los elementos HTML están representados por etiquetas
  • Las etiquetas HTML identifican piezas de contenido, como los encabezados, párrafos, tablas etc
  • Los navegadores no muestran las etiquetas HTML, pero las usan para renderizar el contenido de la pagina

Versiones HTML

Desde los inicios de la web ha habido diferentes versiones de HTML:

  • HTML: 1991
  • HTML2.0: 1995
  • HTML3.2: 1997
  • HTML4.01: 1999
  • XHTML: 2000
  • HTML5: 2014

Elementos HTML

Los elementos HTML consisten usualmente de una etiqueta de inicio y una etiqueta de fin, con el contenido insertado entre estas dos, por ejemplo:

Contenido...]]>

El elemento HTML es todo, del inicio de la etiqueta, hasta el fin.

Los elementos HTML caen en distintas categorías:

  • Raiz
  • Metadatos
  • Seccionado semántico
  • Agrupación de contenido
  • Semántica de texto
  • Edición
  • Contenido embebido
  • Tablas
  • Formularios
  • Scripting

Atributos HTML

  • Todos los elementos HTML pueden tener atributos.
  • Los atributos proveen información adicional acerca de un elemento.
  • Los atributos siempre son especificados en la etiqueta de inicio.
  • Los atributos se declaran en forma de pares: nombre="valor"
Excepto que sea especificado, los atributos pueden tener cualquier cadena de caracteres, incluyendo una vacia. No hay restricción en lo que el texto puede ser, excepto que sea especificado explicitamente.

Los elementos HTML pueden estar anidados, y todos los documentos HTML consisten de elementos anidados,

Se recomienda que los nombres de los atributos sean escritos siempre en letra minúscula. A pesar de que la especificacion HTML5 no requiere comillas dobles alrededor de los valores, se recomienda que siempre se incluyan, ya que son obligatorias en otros tipos de documentos mas estrictos, como XHTML. Aun así, a veces es necesario usar comillas, por ejemplo cuando un valor contiene un espacio.

Atributos globales

Existen ciertos atributos que se pueden aplicar a cualquiera y todos los elementos HTML. Los mas importantes y comunes, son:

  • id
  • class
  • title
  • tabindex
  • style

Atributo id

El atributo id especifica el identificador único del elemento dentro del documento.

El valor debe ser único entre todos los identificadores dentro de su raíz de documento, debe contener al menos un caracter. El valor no puede contener ni un espacio.

No hay restricción alguna en la forma que puede tomar un atributo id, pueden ser dígitos, guiones bajos, tener solo signos de puntuación, etcétera.

El identificador único de un elemento se puede usar para una variedad de propósitos, mas notablemente para ligar partes especificas de un documento, para usar un elemento con scripts, o para asignar un estilo css a un elemento.

Los identificadores son opacos, es decir, no se derivan significados particulares de su valor.

El siguiente código define un atributo y un elemento a que hace liga a dicho elemento: <section id="section1"> Text </section> <a href="#section1">Link to section one</a>

Atributo class

Todos los elementos HTML pueden tener un atributo class especificado.

El atributo, si esta especificado, debe tener un valor que es un conjunto de tokens separados por espacio, que represetan las varias clases a las que el elemento pertenece. Los duplicados son ignorados.

Asignar clases a un elemento, afecta los selectores de CSS, el método getElementsByClassName() en el DOM, etcétera.

No hay restricciones adicionales en los tokens que pueden ser usados en el atributo class, pero se recomienda que se usen valores que describan la naturaleza del contenido, en lugar de valores que representen la presentación deseada del contenido.

El siguiente código define un elemento HTML al que se le define un atributo class; seguido de una porción de código CSS que usa dicha clase:

<a href="./#home" class="home_link">Home</a>
.home_link { display:block; padding:5px; text-decoration:none; color:#333; border-radius:5px; margin: 0px; }

Atributo title

El atributo title representa información adicional del elemento, el cual sería apropiado para una tooltip. En una liga, puede ser el título o description del recurso objetivo; en una imagen, podrían ser los creditos de la imagen o una descripción; en un párrafo, podría ser una nota de pie de página, o comentarios acerca del texto. El valor es texto.

Depender del atributo title no es recomendado, puesto que muchos agentes de usuario no exponen el valor del atributo de una forma accesible requerida por la especificación (requiere un dispositivo como un mouse).

Si el atributo se omite del elemento, se implica que el título del elemento ancestro más cercano que se encuentre definido es relevante para el elemento.

El siguiente código muestra un ejemplo de la etiqueta abbr con un atributo title.

<p>My logs show that there was some interest in <abbr title="HypertextTransport Protocol">HTTP</abbr> today.</p>

Atributo tabindex

El atributo tabindex permite indicar que es posible que un elemento sea enfocable, si se pretende que sea accesible usando navegación secuencial y, en su caso, cual es el orden relativo en la navegación secuencial.

Cuando el atributo es omitido el navegador aplica uno por defecto.

En caso de ser especificado, debe tener un valor que sea un número entero válido, números negativos hacen el elemento inaccesible por navegación secuencial.

El siguiente código muestra el uso del atributo tabindex en un formulario:

<form> <label for="user">Username</label> <input type="text" name="user" tabindex="1" /><br /> <label for="email">e-mail</label> <input type="email" name="email" tabindex="3" /><br /> <label for="phone">Phone number</label> <input type="tel" name="user" tabindex="2" /> </form>

Atributo style

Todos los elementos HTML pueden tener un atributo style. Es un atributo de estilo CSS, definido por la especificacion CSS-STYLE-ATTR

No se debe depender de este atributo, pues algunos navegadores optan por ignorarlo o bloquearlo.

Usar este atributo en particular para mostrar u ocultar contenido es no-estándar, y debe ser evitado. En su lugar se debe usar el attributo hidden.

El código siguiente muestra el uso de el atributo style en un elemento p:

<p style="background-color:#5cf; color:#fff;" >This is a styled paragraph</p>

Entidades HTML

Algunos caracteres son reservados en HTML.

Por ejemplo, si se usa el símbolo menor que (<), o mayor que (>), en texto normal, el navegador los puede interpretar como etiquetas, provocando errores en la renderización.

Para poder mostrar estos caracteres especiales en texto, se usan Entidades HTML.

Las entidades HTML se escriben de la siguiente forma: &nombre_entidad; o bien: &#numero_entidad; Para usar los símbolos mayor que y menor que: &gt; → > &lt; → <

Elementos raíz

Elemento html

El elemento html representa la raiz de los documentos HTML.

Se recomienda incluir el atributo lang en el elemento raiz html, para especificar el lenguaje a usar para motores sintetizadores de voz.

<!DOCTYPE html> <html lang="en"> <head> <title>Example page title</title> </head> <body> <h1>Heading level 1</h1> </body> </html>

Elementos de metadatos

Los metadatos son datos de los datos. Visto de otra forma, es información acerca de la página necesaria para ser mostrada.

Elemento head

El elemento head es el encabezado de la página, éste no es renderizado por el navegador. Se debe incluir como primer elemento dentro del elemento <html>.

Contiene los elementos de metadatos del documento HTML, de los cuales exactamente uno es el elemento <title>, y no más de uno es el elemento <base>.

<!doctype html> <html> <head> <meta charset="utf-8"> <base href="https://www.example.com/"> <title>an application with a long head</title> <link rel="stylesheet" href="default.css"> <link rel="stylesheet alternate" href="big.css" title="big text"> <script src="support.js"></script> <meta name="application-name" content="long headed application"> </head> <body>

Elemento title

El elemento title es un elemento de metadatos. Representa el nombre o título del documento. Se debe usar este elemento para identificar el documento, aun cuando se use fuera de contexto, como en marcadores, historiales o resultados de busqueda.

No puede haber mas que un elemento title en el documento.

El siguiente código muestra un ejemplo de uso del elemento title:

<!doctype html> <html> <head> <title>A document with a short head</title> </head> <body>

Elemento base

El elemento base es un elemento de metadatos.

Permite especificar una URL base de documento, con el proposito de procesar URL's relativas dentro del documento.

Debe tener obligatoriamente un atributo href, un atributo target, o ambos.

El siguiente código muestra un uso correcto del elemento base:

<!doctype html> <html> <head> <title>This is an example for the &lt;base&gt; element</title> <base href="https://www.example.com/news/index.html"> </head> <body> <p>Visit the <a href="archives.html">archives</a>.</p> </body> </html>

Elemento link

El elemento link es un elemento de metadatos.

El elemento link es usado para ligar documentos a otros recursos.

El destino de la liga es deinido por el atributo href, el cual debe estar presente y debe contener una URL válida, no vacía. Si el atributo href no está presente, entonces el elemento no define una liga.

El elemento debe contener un atributo rel, que especifica el tipo de liga.

El siguiente código crea dos ligas a la misma página:

<link rel="author license" href="/about">

Las dos ligas creadas por este elemento poseen una semántica que indica que el recurso ligado contiene información acerca del autor de la página, así como la licencia bajo la que se distribuye su contenido.

Elemento meta

El elemento meta es un elemento de metadatos.

Representa varios aspectos de metadatos que no entran en la categoría, ni pueden ser expresados por los elementos title, base, link, style o script.

El elemento meta puede representar metadatos con el atributo name, directivas pragma con el atributo http-equiv y la declaración de codificación de caracteres con el atributo charset.

El atributo charset no tiene ningún efecto en los documentos XML, solo está permitido para facilitar la migracion hacia y desde XHTML.

No debe haber mas de un elemento meta con un atributo charset por documento.

El siguiente código muestra el uso de un elemento meta, para declarar palabras clave relacionadas con el sitio:

<!doctype html> <html lang="en-GB"> <head> <title>Typefaces on UK motorways</title> <meta name="keywords" content="british,type face,font,fonts,highway,highways"> </head> <body>

Elemento style

El elemento style es un elemento de metadatos.

Permite a los autores incrustar información de estilo en sus documentos. Es una de varias formas de aplicar estilos. El elemento no representa contenido para los usuarios.

El atributo type especifica el lenguaje de estilo. Si el atributo está presente, debe ser un tipo mime válido que designe un lenguaje de estilos. El valor por defecto en caso de que el atributo no esté presente es "text/css".

El siguiente código muestra un ejemplo de un elemento style, con valores que reemplazan el estilo por defecto de a etiqueta em con texto rojo y sin italicas:

<style> em { font-style: normal; color: red; } </style> <p>My <em>favorite</em> book of all time has <em>got</em> to be <cite>A Cat’s Life</cite>. It is a book by P. Rahmel that talks about the <i lang="la">Felis Catus</i> in modern human society.</p>

My favorite book of all time has got to be A Cat’s Life. It is a book by P. Rahmel that talks about the Felis Catus in modern human society.

Elementos para delimitar secciones

Elemento body

El elemento body representa el contenido del documento.

En documentos estándares solo debe existir una etiqueta body.

Contiene todos los elementos que no pertenecen a los metadatos.

Elemento article

El elemento article representa un composición completa e independiente en un documento, página, aplicación o sitio. Puede ser un artículo de una revista, periodico, escolar etc.

El siguiente código es un ejemplo de un elemento article con contenido semanticamente correcto:

<article> <h2><a href="https://herbert.io">Short note on wearing shorts</a></h2> <p>Posted on Wednesday, 10 February 2016 by Patrick Lauke. <a href="https://herbert.io/short-note/#comments">6 comments</a></p> <p>A fellow traveller posed an interesting question: Why do you wear shorts rather than longs? The person was wearing culottes as the time, so I considered the question equivocal in nature, but I attempted to provide an honest answer despite the dubiousness of the questioner’s dress.</p> <p>The short answer is that I enjoy wearing shorts, the long answer is...</p> <p><a href="https://herbert.io/short-note/">Continue reading: Short note on wearing shorts</a></p> </article>

Elemento section

El elemento section representa una sección de un documento o aplicación. Una sección, en este contexto es una agrupación tematica de contenido. Cada sección debe ser identificada incluyendo al menos un elemento h1-h6 dentro de su contenido.

El elemento section no es un contenedor generico de contenido. Cuando se desee un contenedor solo para propósitos de estilo se recomienda usar un elemento div.

El siguiente código muestra dos secciones, cada una con su propio contenido, pero aun asi relacionadas entre sí:

<section> <h3>Elemento <a href="https://www.w3.org/TR/html/sections.html#the-nav-element" target="_blank"><code>section</code></a></h3> <hr /> </section> <section> <h3>Elemento <a href="https://www.w3.org/TR/html/sections.html#the-nav-element" target="_blank"><code>nav</code></a></h3> <hr /> </section>

Elemento nav

El elemento nav representa una sección de una página que contiene ligas a otras páginas o partes dentro de la mísma página: una sección con ligas de navegación.

No todos los grupos de ligas de navegación tienen que ir dentro de un elemento nav, el elemento esta destinado primariamente a bloques de navegacion mayores.

<nav> <a href="./index.xhtml">Home</a> <a href="./miku.xhtml">初音ミク</a> <a href="./music.xhtml">Music</a> <a href="./art.xhtml">Art</a> <a href="./links.xhtml">Links</a> <a href="./htmltut.xhtml">Tutorial HTML</a> </nav>

Elementos h1, h2, h3, h4, h5 y h6

Los elementos h# representan encabezados para sus secciones.

Estos elementos tienen un rango, dado por el numero en su nombre. El elemento h1 tiene el rango más alto, el elemento h6 tiene el rango mas bajo, y dos elementos con el mismo nombre tienen el mismo rango.

Se usa el rango de los encabezados para crear la estructura del documento.

El siguiente código muestra un documento con 6 niveles de encabezados:

<h1>top level heading</h1> <section><h2>2nd level heading</h2> <section><h3>3nd level heading</h3> <section><h4>4th level heading</h4> <section><h5>5th level heading</h5> <section><h6>6th level heading</h6> </section> </section> </section> </section> </section>

Elemento header

El elemento header representa contenido introductorio para su ancestro mas cercano de seccionado. Contiene usualmente un grupo de ligas introductorias o ayuda navegacional.

El siguiente código muestra un encabezado de ejemplo:

<header> <p>Welcome to...</p> <h1>Voidwars!</h1> </header>

Elemento footer

El elemento footer representa un pie de página para su ancestro mas cercano de seccionado. Usualmente contiene información acerca de la sección, como quien la escribió, información de copyright o ligas a documentos relacionados.

La información de contacto pertenece al elemento address, posiblemente dentro de un elemento footer.

El siguiente código muestra un ejemplo de un elemento footer:

<footer>&#169;2017 </footer>

Elemento address

Elemento address representa la información de contacto de los elementos ancestro mas cercanos de tipo article o body, si el elemento body es el más cercano, entonces la información de contacto aplica al documento entero.

<address> <A href="../People/Raggett/">Dave Raggett</A>, <A href="../People/Arnaud/">Arnaud Le Hors</A>, contact persons for the <A href="Activity">W3C HTML Activity</A> </address>

Elementos para agrupar contenido

Elemento p

El elemento p representa un párrafo.

En HTML un párrafo no es un concepto lógico, sino estructural. Para usar el significado lógico de un párrafo, se recomienda usar el elemento div en lugar del elemento p.

El elemento p no debe ser utilizado cuando haya un elemento específico más apropiado.

El siguiente código muestra un ejemplo del uso del elemento p:

<p>There was once an example from Femley,<br /> Whose markup was of dubious quality.<br /> The validator complained,<br /> So the author was pained,<br /> To move the error from the markup to the rhyming.</p>

Demostración del código anterior:

There was once an example from Femley,
Whose markup was of dubious quality.
The validator complained,
So the author was pained,
To move the error from the markup to the rhyming.

Elemento hr

El elemento hr representa un salto de parrafo temático, es decir un cambio de tema dentro de una sección.

<p>Radios use the electromagnetic spectrum in the meter range and longer.</p> <hr /> <p>Signal flares use the electromagnetic spectrum in the nanometer range.</p>

Radios use the electromagnetic spectrum in the meter range and longer.

Signal flares use the electromagnetic spectrum in the nanometer range.

Elemento pre

El elemento pre represeta un bloque de texto pre-formateado, cuya estructura está definida por convenciones tipográficas, en lugar de los elementos.

Algunos usos del elemento pre son:

  • Incluir un e-mail, con párrafos indicados por saltos de linea, lineas de listas con viñetas
  • Incluir fragmentos de código de computadora, estructurado de acuerdo a las convenciones de su lenguaje.
  • Mostrar arte ASCII
<p>This is the <code>Panel</code> constructor:</p> <pre><code>function Panel(element, canClose, closeHandler) { this.element = element; this.canClose = canClose; this.closeHandler = function () { if (closeHandler) closeHandler() }; }</code></pre>

This is the Panel constructor:

function Panel(element, canClose, closeHandler) {
  this.element = element;
  this.canClose = canClose;
  this.closeHandler = function () { if (closeHandler) closeHandler() };
}

Elemento blockquote

El elemento blockquote representa contenido que es citado de otra fuente, opcionalmente con una cita que debe estar en un elemento footer o cite.

<figure> <blockquote> "That monster custom, who all sense doth eat Of habit’s devil," <abbr title="et cetera">&c.</abbr> not in Folio "What a falling off was there ! From me, whose love was of that dignity That it went hand in hand even with the vow I made to her in marriage, and to decline Upon a wretch." </blockquote> <footer> — <cite class="title">Shakespeare manual</cite> by <cite class="author">Frederick Gard Fleay</cite>, p19 (in Google Books) </footer> </figure>

Elemento ol

El elemento ol representa una lista de elementos intencionalmente ordenados, es decir, cambiar el orden de los elementos cambiaria el significado del contenido.

Los elementos de la lista son los elementos li en orden de árbol.

<p>I have lived in the following countries (given in the order of when I first lived there):</p> <ol> <li>Switzerland</li> <li>United Kingdom</li> <li>United States</li> <li>Norway</li> </ol>

I have lived in the following countries (given in the order of when I first lived there):

  1. Switzerland
  2. United Kingdom
  3. United States
  4. Norway

Elemento ul

El elemento ul representa una lsita de elementos, donde el orden de éstos no es importante, es decir, cambiar el orden de los elementos no cambiaria el significado del contenido.

Los elementos de la lista son los elementos li en orden de árbol.

<p>I have lived in the following countries:</p> <ul> <li>Norway</li> <li>Switzerland</li> <li>United Kingdom</li> <li>United States</li> </ul>

I have lived in the following countries:

  • Norway
  • Switzerland
  • United Kingdom
  • United States

Elemento li

El elemento li representa un elemento de una lista. Sus elementos padre pueden ser ol, ul o menu.

<figure> <figcaption>The top 10 movies of all time</figcaption> <ol> <li value="10"><cite>Josie and the Pussycats</cite>, 2001</li> <li value="9"><cite lang="sh">Црна мачка, бели мачор</cite>, 1998</li> <li value="8"><cite>A Bug’s Life</cite>, 1998</li> <li value="7"><cite>Toy Story</cite>, 1995</li> <li value="6"><cite>Monsters, Inc</cite>, 2001</li> <li value="5"><cite>Cars</cite>, 2006</li> <li value="4"><cite>Toy Story 2</cite>, 1999</li> <li value="3"><cite>Finding Nemo</cite>, 2003</li> <li value="2"><cite>The Incredibles</cite>, 2004</li> <li value="1"><cite>Ratatouille</cite>, 2007</li> </ol> </figure>

Elemento dl

El elemento dl representa una lista de descripción de cero o mas grupos de terminos de descripción.

<dl> <dt>Blanco tequila</dt> <dd>The purest form of the blue agave spirit...</dd> <dt>Reposado tequila</dt> <dd>Typically aged in wooden barrels for between two and eleven months...</dd> </dl>

Elemento figure

El elemento figure representa un contenido, opcionalmente con una acotación, que es independiente, y es frecuentemente referenciada como una unidad que forma parte del contenido del documento.

Este elemento puede ser usado entonces para anotar imágenes, diagramas, fotos, fragmento de código, etc.

In listing 4 we see the primary core interface API declaration.

Elemento figcaption

El elemento figcaption representa una leyenda o subtítulo, para el contenido del elemento figure padre, de existir.

<figure> <p>'Twas brillig, and the slithy toves<br /> Did gyre and gimble in the wabe;<br /> All mimsy were the borogoves,<br /> And the mome raths outgrabe.</p> <figcaption><cite>Jabberwocky</cite> (first verse). Lewis Carroll, 1832-98</figcaption> </figure>

Elemento div

El elemento div no tiene un significado especifico del todo. Representa a sus elementos hijo. Es usado para marcar semanticas con los atributos title, class y lang, en un grupo de elementos consecutivos.

<article lang="en-US"> <h2>My use of language and my cats</h2> <p>My cat’s behavior hasn’t changed much since her absence, except that she plays her new physique to the neighbors regularly, in an attempt to get pets.</p> <div lang="en-GB"> <p>My other cat, colored black and white, is a sweetie. He followed us to the pool today, walking down the pavement with us. Yesterday he apparently visited our neighbours.
